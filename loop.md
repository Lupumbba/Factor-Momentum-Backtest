# 动量选股月度循环

## 1. 目标

这个文件解决一个很具体的问题：

> 每个月根据当前动量策略，我应该优先研究哪些股票、保留哪些股票、减掉哪些股票，以及如何完成调仓？

本项目基于并扩展自
[Xuanyu6Chen/Factor-Momentum-Backtest](https://github.com/Xuanyu6Chen/Factor-Momentum-Backtest)。

原始项目提供 12-1 动量因子的回测基础。本版本在此基础上增加了自动选股、保守入场过滤、交易成本压力测试和本地可视化页面，让它更适合作为月度选股和调仓流程使用。

注意：这个流程是系统化研究工具，不是投资建议，也不是自动买入保证。

## 2. 核心思路

按照你和 ChatGPT 对话里的最后建议，当前项目先按三个层次推进：

1. ETF 双动量：稳定版，用 ETF 做资产轮动。
2. 美股强势股动量：进攻版，用股票池做 12-1 动量选股。
3. 低波动质量动量：防守版，用低波动、趋势和基本面质量筛出更稳的股票候选。

股票动量部分每个月只问三个问题：

1. 动量是否足够强：股票是否排在 12-1 动量排名前列？
2. 趋势是否仍然健康：当前价格是否站上 200 日均线？
3. 风险是否过高：最近波动率是否没有处在股票池中最高的一档？

只有同时满足这三点的股票，才进入“优先研究”候选池。

ETF 双动量部分每个月只问两个问题：

1. SPY、QQQ、IWM、TLT、GLD 中谁的 6 个月趋势最强？
2. 如果最强资产的 6 个月收益也不是正的，是否应该切到 SHY 防守？

低波动质量动量部分每个月只问四个问题：

1. 股票是否仍有正的 12-1 动量？
2. 股票是否仍站上 200 日均线？
3. 最近 126 个交易日波动是否没有过高？
4. 当前基本面质量分是否过线？

## 3. 每月执行时间

建议在每个月结束后执行一次。

最简单的节奏：

1. 等到月末结束后的第一个交易日。
2. 重新运行项目，更新最新数据和报告。
3. 打开本地网站，看最新“优先研究”名单。
4. 把新名单和自己当前持仓做对比。
5. 按规则决定新增、保留、减仓、移除。
6. 记录本月决策，下个月重复。

不要在一个月中频繁改动，否则月度动量策略会变成短线追涨。

## 4. 运行项目

在项目根目录执行：

```bash
source .venv/bin/activate
PYTHONPATH=Src python -m factor_momentum.pipeline
open Results/site/index.html
```

主要看这几个输出：

| --- | --- |
| `Results/site/index.html` | 最主要的可视化页面，优先看这个 |
| `Results/site/etf_dual_momentum/index.html` | ETF 双动量稳定版看板 |
| `Results/site/low_vol_quality_momentum/index.html` | 低波动质量动量防守版看板 |
| `Results/site/large_cap_100/index.html` | 扩展大盘 100 只股票池页面 |
| `Results/site/sector_balanced_100/index.html` | 行业均衡 100 只股票池页面 |

| `Results/latest_selection.csv` | 最新自动选股结果 |

## 4.1 每月先看 ETF 稳定版

ETF 双动量看板回答的是：

> 当前应该偏向股票、债券、黄金，还是短债防守？

打开：

```bash
open Results/site/etf_dual_momentum/index.html
```

默认规则：

1. 每月比较 `SPY`、`QQQ`、`IWM`、`TLT`、`GLD` 的 6 个月收益。
2. 选择 6 个月收益最高的资产。
3. 如果最高收益大于 0，则持有这个资产。
4. 如果最高收益小于等于 0，则切换到 `SHY`。
5. 每月调仓一次。

这个页面更适合作为“市场状态判断器”。如果它已经切到防守，股票动量组合也应该降低仓位，而不是继续激进加仓。

## 4.2 每月再看低波动质量防守版

低波动质量动量看板回答的是：

> 如果我不想只追最强股票，而是想找更稳的强势股，本月应该优先研究哪些？

打开：

```bash
open Results/site/low_vol_quality_momentum/index.html
```

默认规则：

1. 从 `sector_balanced_100` 股票池开始。
2. 先筛出 12-1 动量为正的股票。
3. 再要求价格站上 200 日均线。
4. 再剔除近期 126 个交易日波动过高的股票。
5. 对剩下股票按“动量 + 低波动”排序。
6. 抓取最新基本面质量快照，给盈利能力、现金流收益和负债压力打分。
7. 最终候选必须同时通过价格规则和质量规则。
8. 最多选 8 只；如果候选少于 5 只，剩余仓位保留现金。

重要限制：

历史回测只验证价格规则，不会把今天看到的基本面数据回填到过去。最新候选名单会叠加当前基本面质量，这样更适合做当月研究清单。

## 4.3 如何选择股票池

现在项目会同时跑三个股票池：

| 股票池 | 含义 | 适合怎么用 |
| --- | --- | --- |
| `author_50` | 作者原始 50 只 | 作为基准，不轻易删除 |
| `large_cap_100` | 扩展大盘 100 只 | 看更宽股票池是否带来更好的动量机会 |
| `sector_balanced_100` | 行业均衡 100 只 | 看降低行业集中度后，回撤和 Sharpe 是否更稳 |

每个月先看 `Results/site/index.html` 里的“股票池对比”：

1. 如果某个股票池 Sharpe 更高，同时最大回撤没有明显恶化，可以优先研究它。
2. 如果某个股票池累计收益更高，但回撤也明显更深，不要只因为收益高就切过去。
3. 如果行业均衡版本收益略低但回撤更浅，可以作为更稳健的参考组合。
4. 不要每个月在三个股票池之间频繁切换，至少连续观察 3 个月以上再改变主参考股票池。

## 5. 如何理解自动选股结果

`Results/latest_selection.csv` 里最重要的是这些字段：

| 字段 | 含义 | 使用方式 |
| --- | --- | --- |
| `signal_date` | 本次信号日期 | 确认数据是否已经更新到最新 |
| `ticker` | 股票代码 | 被分析的股票 |
| `rank` | 动量排名 | 数字越小，动量越强 |
| `mom12_1` | 12-1 动量得分 | 越高代表过去 12 个月、剔除最近 1 个月后的趋势越强 |
| `above_sma` | 是否站上 200 日均线 | 用来确认中长期趋势 |
| `high_volatility` | 最近波动率是否偏高 | 用来识别追高和剧烈波动风险 |
| `strategy_signal` | 原始策略是否选中 | 代表它进入了动量前列 |
| `conservative_entry_signal` | 是否通过保守入场过滤 | 这是优先研究名单 |
| `conservative_equal_weight` | 保守候选股的等权参考仓位 | 可作为初始目标仓位参考 |
| `reason` | 通过或失败原因 | 用来判断新增、保留、减仓、移除 |

优先级从高到低：

1. `conservative_entry_signal = True`：优先研究或持有。
2. `strategy_signal = True` 但 `conservative_entry_signal = False`：动量强，但风险或趋势不够好，只观察或减仓。
3. `strategy_signal = False`：不在本月动量组合里，原则上移出候选池。

## 5.1 如何理解防守版结果

低波动质量动量页面里最重要的是这些字段：

| 字段 | 含义 | 使用方式 |
| --- | --- | --- |
| `defensive_entry_signal` | 是否进入防守版候选 | True 才是本月优先研究名单 |
| `target_weight` | 单只股票参考仓位 | 默认最多 8 只，通常等权 |
| `cash_weight` | 现金预留 | 候选股不足时不要硬凑满仓 |
| `final_score` | 综合分 | 价格规则和质量规则合成后的排序分 |
| `quality_score` | 基本面质量分 | 越高代表盈利能力、现金流和负债压力整体更好 |
| `momentum_12_1` | 12-1 动量 | 必须为正才进入价格规则 |
| `annualized_volatility` | 年化波动 | 防守版会避开波动过高的股票 |
| `reason` | 通过或失败原因 | 用来判断是质量不足、趋势破坏还是波动过高 |

这个页面更适合作为“防守仓位参考”。如果 ETF 双动量已经偏防守，优先看这个页面，而不是只看进攻版股票动量。

## 6. 默认组合构建规则

默认只把 `conservative_entry_signal = True` 的股票放进核心候选池。

建议规则：

1. 候选股数量大于等于 5 只：所有候选股等权配置。
2. 候选股数量少于 5 只：单只股票最高 20%，剩余资金保留现金。
3. 不给 `high_volatility = True` 的股票新增资金。
4. 不给 `above_sma = False` 的股票新增资金。
5. 不为了满仓而强行买入不符合规则的股票。

这样可以让组合尽量贴合动量因子，同时避免过度集中在一两只股票上。

## 7. 每月调仓规则

每个月把“当前持仓”和“最新 selection 表”做对比。

### 新增

满足以下条件才新增：

- `conservative_entry_signal = True`
- 当前没有持有
- 加入后不超过单只股票仓位上限

目标仓位：

- 候选股 1 到 4 只：每只最多 20%，剩余保留现金。
- 候选股 5 只或以上：每只仓位为 `1 / 候选股数量`。

### 保留

满足以下条件就保留：

- 当前已经持有
- `conservative_entry_signal = True`

如果当前仓位和目标仓位差距不大，可以不调，避免交易成本过高。

### 减仓

满足以下条件时，停止新增并考虑减仓：

- `strategy_signal = True`
- 但 `conservative_entry_signal = False`

这说明股票仍然有较强动量，但趋势过滤或波动过滤没有通过。它可以继续观察，但不应该作为保守组合的新资金入口。

### 移除

满足以下任一条件时，从组合里移除：

- `strategy_signal = False`
- `above_sma = False`
- `high_volatility = True`

移除规则的作用是让组合始终贴合当前动量和风险条件，而不是因为过去曾经强势就一直持有。

## 8. 交易成本检查

调仓前看网站里的“交易成本压力测试”。

一般这样理解：

| 成本 | 含义 |
| --- | --- |
| 0 bps | 理想情况，不考虑交易摩擦 |
| 10 bps | 常规基准假设 |
| 50 bps | 压力测试，模拟更高交易成本 |

如果策略只在 0 bps 下很好，但到了 50 bps 明显变差，就不要频繁调仓。此时只处理信号变化很明确的股票。

## 9. 市场趋势检查

网站里的策略结果是第一层筛选，不代表可以无条件买入。

每个月至少检查：

1. 策略净值曲线是否仍然整体向上？
2. 当前回撤是否明显扩大？
3. 加入交易成本后，Sharpe 是否仍然可接受？
4. 候选股是否集中在同一个行业？
5. 候选股是否普遍波动很大？

如果风险明显升高，应该降低总仓位，而不是为了满仓强行买入。

## 11. 标准月度流程

每个月照这个顺序做：

1. 运行完整 pipeline。
2. 先打开 `Results/site/etf_dual_momentum/index.html`，判断本月市场是进攻还是防守。
3. 再打开 `Results/site/low_vol_quality_momentum/index.html`，查看防守版候选。
4. 最后打开 `Results/site/index.html`，查看进攻版股票动量名单。
5. 如果 ETF 页面偏防守，降低进攻版权重，优先研究防守版候选。
6. 如果 ETF 页面仍持有风险资产，再用进攻版和防守版名单做组合搭配。
7. 新增新的候选股。
8. 保留仍然通过过滤的持仓。
9. 减少动量仍强但风险过滤未通过的股票。
10. 移除已经不在动量信号中、跌破 200 日均线、波动过高、或质量不过线的股票。
11. 按默认组合构建规则调整仓位。
12. 填写本月记录模板。

## 12. 下一步可以增强什么

当前已经有三套策略看板：

1. ETF 双动量：判断大类资产方向。
2. 股票 12-1 动量：找进攻型强势股。
3. 低波动质量动量：找防守型强势股。

后续最值得增强的是：

1. 增加行业集中度限制。
2. 增加单只股票最大仓位限制。
3. 增加最小调仓阈值，避免为了很小的仓位差频繁交易。
4. 导出每月调仓清单。
5. 接入真正 point-in-time 的历史基本面数据库，让质量因子也可以进入无未来函数的历史回测。

完成这些后，这个项目就会更接近一个完整的动量因子组合管理工具。
